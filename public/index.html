A bit about GraphQL

One of the benefits of using GraphQL (over REST) is that it gives you an amazing UI that lets you test out the queries that you make.

There are other, more important, reasons for using it; for example, it allows for queries (from the client-side) to retrieve specific bits of data without the need for creating new routes.

But, I just used it because I liked how it looked.

Basic usage

Now I won't be able to explain this very well, but I will give you some pointers on how to use the UI:

  1. Write everything inside a pair of curly brackets.
    For example a query might look like this:
    
{
  theColourOfTheSky {
    colour
  }
}
    The curly brackets after the 'theColourOfTheSky' query is what you want as a response (a colour).
  2. To have a look at all the queries you can run, click on the DOCS tab on the right.
